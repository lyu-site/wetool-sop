威拓資料夾三層級架構

公司檔案依存放位置分為三個層級，各自有明確的定位與存取規範，請依規定存放。

🗄️
核心端（NAS）
嚴格管控 · 完整保存
  • 正式專案資料
  • BIM 資源庫
  • 威拓數位資產庫
☁️
共享雲端（Google）
限制範圍 · 對外窗口
  • 業主共享資料區
  • 正式成果提交
  • 跨公司協作
  • 外部收件
  • 雲端程式連結放置（例：名片管理）
💻
個人端（PC/Notion）
開放自主 · 定期同步
  • 個人工作流程
  • 備份工作中的專案檔與圖說
💡 核心原則 正式資料一律存入 NAS 核心端。個人端僅作為工作暫存，完成後需同步回 NAS。Google 雲端僅用於對外交付，不作為主要儲存位置。

圖說收文資料夾命名規範

收文記錄資料夾的命名直接影響 Python 腳本能否正確抓取日期和來源，請嚴格遵守以下格式。

命名格式

YYYYMMDD_管道_來源_類型
20260412_LINE_楊主任_建築施工 ← 日期_管道_來源_類型
20260412_LINE_何協理_景觀+公設 ← 多類型用 + 連接
20260415_Email_業主_建築+結構建照
20260415_現場_楊主任_結構施工

各欄位說明

欄位說明範例
YYYYMMDD收文日期，8碼數字，必填
腳本從這裡抓取「收文日期」欄位		20260412
管道收到圖說的方式LINE Email 現場 Google雲端
來源提供圖說的人或單位楊主任 何協理 業主
類型圖說的專業類型，多類型用 + 連接建築 結構 機電 景觀 全專業
⚠️ 最重要的規則 資料夾名稱最前面必須為日期，腳本依靠這個規則抓取日期。 ⚠️ 開頭 8 碼數字 YYYYMMDD若日期格式錯誤，Excel 的收文日期欄位將會空白，將影響後續平台抓取最新圖說與套用圖說資訊。

腳本行為說明

情況腳本行為
第一次執行抓取所有 CAD 連結，套用日期 = 今天，寫入平台
再次執行（舊連結）保留原本套用日期，不覆蓋
新增 CAD 連結後執行只有新的那筆套用日期 = 今天，舊的不變
手動在平台填的資料執行後保留，不會被覆蓋
📌 新專案部署時需修改 Dynamo 腳本路徑 在 Python Script 節點裡找到這行並修改：
DATA_JSON = r"\NAS位址\...\新專案\01_圖說管理\【圖說管理平台】\data.json"

常見問題排解

問題原因解決方式
.bat 雙擊跳一下就消失bat 檔編碼錯誤用記事本開啟 → 另存新檔 → 編碼選 ANSI → 換行選 Windows (CRLF) → 存檔類型「所有檔案」→ 副檔名 .bat
同事打開網址顯示「無法連線」伺服器未啟動或視窗被關確認負責人電腦上「啟動伺服器」視窗是否還開著
新圖說存進資料夾後平台沒更新自動化視窗被關，或還未到 60 秒確認「啟動收文自動化」視窗是否還開著，等待最多 60 秒後按 F5
平台左下角顯示紅點伺服器連線中斷重新雙擊「啟動伺服器.bat」，再重新整理網頁
圖說資料重複出現.processed_files.txt 遺失新版 run.py 有雙重防護，重新啟動自動化腳本即可。若仍重複，執行「清理重複資料.py」
編輯圖說儲存後沒有變化資料有重複 id執行「清理重複資料.py」修復 data.json
批次修改只改到一筆同上，重複 id 問題執行「清理重複資料.py」修復 data.json

新專案部署說明

開始一個新專案時，只需要複製現有資料夾並修改幾個路徑設定。

1
在 NAS 建立新專案資料夾結構
新專案\
└── 01_圖說管理\
    ├── 01_業主提供\01_收文記錄\
    ├── 02_整理圖說\
    ├── 【圖說收文自動化】\
    └── 【圖說管理平台】\
2
複製平台檔案
複製哪個檔案放到哪裡
server.py新專案 【圖說管理平台】\
index.html新專案 【圖說管理平台】\
啟動伺服器.bat新專案 【圖說管理平台】\
run.py新專案 【圖說收文自動化】\
啟動收文自動化.bat新專案 【圖說收文自動化】\
⚠️ 不要複製這些檔案 data.json、.processed_files.txt、福祿段_收文紀錄.xlsx 都是舊專案的資料
3
修改 run.py 的路徑設定
# ★ 路徑設定區（只需要改這裡）★

WATCH_FOLDER = r"\\WETOOL\wt\...\新專案\01_圖說管理\01_業主提供\01_收文記錄"
EXCEL_PATH = r"\\WETOOL\wt\...\新專案\01_圖說管理\【圖說收文自動化】\新專案_收文紀錄.xlsx"
DATA_JSON = r"\\WETOOL\wt\...\新專案\01_圖說管理\【圖說管理平台】\data.json"
PROCESSED_LOG = r"\\WETOOL\wt\...\新專案\01_圖說管理\【圖說收文自動化】\.processed_files.txt"
4
修改兩個 bat 檔的路徑
用記事本開啟，分別修改 net use 那行路徑，換成新專案對應資料夾。存檔編碼選 ANSI，換行選 Windows (CRLF)
5
修改平台的專案名稱
開啟平台網頁後，左側點「專案設定」→ 修改專案名稱、業主名稱、負責人 → 儲存
✅ 總結：新專案只需要改 4 個地方 run.py 的 4 行路徑 + 啟動伺服器.bat 的 1 行 + 啟動收文自動化.bat 的 1 行 + 平台裡的專案設定
